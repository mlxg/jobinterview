YAHOO Performance

Content
  1. Make Fewer HTTP Requests  -  尽量减少http请求
  2. Reduce DNS Lookups  -  减少DNS查找次数 
  3. Avoid Redirects  -  避免重定向  
  4. Make Ajax Cacheable  -  Ajax 缓存
  5. Postload Components  -  延迟加载
  6. Preload Components  -  预加载
  7. Reduce the Number of DOM Elements  -  减少DOM元素数量
  8. Split Components Across Domains  -  跨域分离组件
  9. Minimize Number of iframes  -  减少使用iframe
  10. Avoid 404s  -  避免404

Server
  1. Use a Content Delivery Network (CDN)  -  使用CDN内容分发
  2. Add Expires or Cache-Control Header  -  添加 Expires or Cache-Control 头
  3. Gzip Components  -  使用Gzip压缩 (修改http服务器配置 nginx, apache, node, iis)
  4. Configure ETags  -  配置 ETags
  5. Flush Buffer Early  -  早数据缓存
  6. Use GET for Ajax Requests  -  使用 Ajax 请求
  7. Avoid Empty Image src  -  避免无效的图片连接
  8. Keep alive  - 就是浏览器和服务器之间保持长连接,这个连接是可以复用的,当客户端发送一次请求后,收到相应内容后,这个连接会保持一段时间,在该时间内的第二次请求就不需要再重新建立连接.会增加服务器压力(静态建议开启,动态建议关闭)

Cookie

  1. Reduce Cookie Size  -  减少Cookie 大小
  2. Use Cookie-Free Domains for Components  -  合理使用cookie


Css
  1. Put Stylesheets at Top  -  把样式表放在顶部
  2. Avoid CSS Expressions  -  避免使用css Expressions属性
  3. Choose <link> Over @import  -  尽量使用link 而不是@import
  4. Avoid Filters  - 避免使用Filters
Javascript
  1. Put Scripts at Bottom  -  把脚本放在页面底部
  2. Make JavaScript and CSS External  -  使用外部Js 和 Css
  3. Minify JavaScript and CSS  -  压缩Js 和 Css
  4. Remove Duplicate Scripts  -  删除重复的脚本
  5. Minimize DOM Access  - 减少DOM操作
  6. Develop Smart Event Handlers  -  基于事件
Images
  1. Optimize Images  -  优化图片
  2. Optimize CSS Sprites  -  优化css Sprites
  3. Do Not Scale Images in HTML  -  使用对应的图片, 而不是在html scale 图片
  4. Make favicon.ico Small and Cacheable  -  favicon.ico 尽量小 且 使用缓存
Mobile
  1. Keep Components Under 25 KB  -  让组件保持在25kb 以下, iPhone将超过25k或更大的组件 不会使用高速缓存
  2. Pack Components Into a Multipart Document  -  包组件集成到一个多文档
